Rock the Vote: Advanced Features

Assignment

Overview

In this final phase, you’ll enhance your app with security upgrades, protected UX flows, and voting. Implement the items below across both backend and frontend to complete Part 3 of Rock the Vote.

Backend Enhancements

Updated User Model

  • Password Hashing: Use bcrypt in a pre-save hook to hash passwords before storing.
  • Password Removal: Add userSchema.methods.withoutPassword() to strip the password prior to sending user data to clients or signing tokens.

Updated Login and Signup Routes

  • Signup: Hash the password, save the user, return { user: withoutPassword(), token }.
  • Login: Verify the password with bcrypt.compare(), return { user: withoutPassword(), token }.

Frontend Enhancements

Conditional Rendering

  • Show different UI when a token is present (authenticated) vs. absent (guest).
  • Persist user and token in localStorage; initialize context from storage on load.

Protected Routes

  • Use React Router to gate pages that require authentication.
  • Redirect unauthenticated users to the login page.

Voting Functionality

Upvotes & Downvotes

  • Add upvote/downvote buttons to each Issue component.
  • On click, send a request to update the issue’s votes and optimistically update UI state.
  • Display current vote totals and sort the feed by upvotes (descending).
Example Snippets (Optional)

Server route (concept):

issueRouter.put('/upvotes/:issueId', async (req, res, next) => {
  try {
    const updated = await Issue.findByIdAndUpdate(
      req.params.issueId,
      { $addToSet: { upvotes: req.auth._id }, $pull: { downvotes: req.auth._id } },
      { new: true }
    );
    res.status(201).send(updated);
  } catch (err) { res.status(500); next(err); }
});

Client sort (concept):

const sorted = [...allIssues].sort((a, b) => (b.upvotes?.length || 0) - (a.upvotes?.length || 0));
