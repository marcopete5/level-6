Importance of Displaying Conditional User Actions and Keeping User Data Secure

Conditional rendering of user-specific actions (like Edit/Delete) improves UX by showing only relevant options, reduces clutter, and enforces access control in the UI. Pair this with server-side authorization to protect privacy, prevent unauthorized changes, and maintain data integrity.

  • UX: Relevant, empowering, and efficient interactions.
  • Security: Only owners can modify/delete their content.
  • Privacy & Compliance: Protects sensitive data and supports regulatory obligations.

Client-Side Conditional Rendering (React) 


import { useUser } from "../context/UserContext"

function IssueItem({ issue, onEdit, onDelete }) {
  const { userState: { user } } = useUser()
  const isOwner = String(issue.userId) === String(user?._id)

  return (
    <article className="card">
      <h4>{issue.title}</h4>
      <p>{issue.description}</p>

      {isOwner && (
        <div className="btn-row">
          <button className="btn-secondary" onClick={() => onEdit(issue)}>Edit</button>
          <button className="btn-danger" onClick={() => onDelete(issue._id)}>Delete</button>
        </div>
      )}
    </article>
  )
}

Note: The client should hide actions for non-owners, but security must be enforced on the server.

Server-Side Authorization (Express + Mongoose) 


const express = require("express")
const Issue = require("../models/Issue")
const issueRouter = express.Router()

// UPDATE (owner only)
issueRouter.put("/:issueId", async (req, res, next) => {
  try {
    const updated = await Issue.findOneAndUpdate(
      { _id: req.params.issueId, userId: req.auth._id }, // enforce ownership
      req.body,
      { new: true }
    )
    if (!updated) {
      res.status(403)
      return next(new Error("Not authorized to update this issue"))
    }
    res.status(200).send(updated)
  } catch (err) {
    res.status(500)
    next(err)
  }
})

// DELETE (owner only)
issueRouter.delete("/:issueId", async (req, res, next) => {
  try {
    const deleted = await Issue.findOneAndDelete({
      _id: req.params.issueId,
      userId: req.auth._id // enforce ownership
    })
    if (!deleted) {
      res.status(403)
      return next(new Error("Not authorized to delete this issue"))
    }
    res.status(200).send({ msg: "Issue removed" })
  } catch (err) {
    res.status(500)
    next(err)
  }
})

module.exports = issueRouter

Pair UI checks with authorization filters (e.g., { _id, userId: req.auth._id }) so non-owners cannot forge requests.
